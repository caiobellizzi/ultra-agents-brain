# Relabel Agno dashboard provider from OpenAI → LiteLLM

## Context

Every agent in `ultra-agents-brain` routes through the LiteLLM proxy on
`:4000` using Agno's `OpenAIChat` class. Agno introspects the model class
to populate the dashboard config view at `os.agno.com/config?type=agent`,
showing `name: OpenAIChat` / `provider: OpenAI` for all agents — visually
misleading now that requests fan out to NVIDIA NIM, LM Studio, Anthropic,
and Groq behind the proxy.

`OpenAIChat` (at `.venv/lib/python3.13/site-packages/agno/models/openai/chat.py:34`)
defines two class attributes that drive the dashboard labels:

```python
class OpenAIChat(Model):
    id: str = "gpt-5.4-mini"
    name: str = "OpenAIChat"
    provider: str = "OpenAI"
```

Both are plain class attrs — overriding them in a subclass changes the
dashboard rendering without touching call semantics.

## Decisions

| # | Decision | Resolution |
|---|---|---|
| Q1 | Subclass or monkey-patch? | **Subclass.** Cleaner, no global side-effect, self-documenting. |
| Q2 | New file or inline in `model.py`? | **Inline.** ~10 lines, single use-site, keeps the factory and label co-located. |
| Q3 | Label string | `name = "LiteLLM"`, `provider = "LiteLLM"`. Truthful (everything traverses LiteLLM); fan-out behind LiteLLM is documented in README. |
| Q4 | Touch the Agno `provider in ["Nvidia", ...]` tool-stripping branch? | **No.** Phase 08 smoke confirmed current calls succeed without that strip path. Out of scope. |

## Tasks

### Task 1 — `agentos/model.py`: add `LiteLLMChat` subclass, switch factory

Inline subclass at module scope (after the import, before `_TIER_ENV`):

```python
class LiteLLMChat(OpenAIChat):
    """OpenAIChat against the LiteLLM proxy — relabels dashboard provider."""
    name: str = "LiteLLM"
    provider: str = "LiteLLM"
```

Change the factory return to `LiteLLMChat(...)`.
