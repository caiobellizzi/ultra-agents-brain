# Wave 4 — Telegram Adapter + Vault Reindex

## Objective

Update `channels/telegram_adapter.py` to consume typed responses (extract fields from `ChatReply`, `QueryAnswer`, etc.) and document the one-shot vault reindex command. This is the final integration wave before evals.

## Context

The Telegram adapter currently does string extraction on AgentOS JSON responses. After Wave 3, agents return typed Pydantic objects serialized as JSON. The adapter needs minor updates to extract the right fields (`text`, `citations`, `answer`) rather than treating the whole JSON as a message body.

The HITL flow (`/continue` endpoint handling) must not change — it is already correct per v1.0 requirements.

Vault reindex is a one-shot operation needed after Wave 3 deploys: `python -m agentos.knowledge --reindex` loads all vault `.md` files into `agno_knowledge` PgVector table.

## Tasks

### 1. Update response extraction in `channels/telegram_adapter.py`

Locate the response text extraction logic (typically something like `data["output"]` or `data["content"]`). Update to handle structured agent responses:

```python
def extract_reply_text(agent_response: dict) -> str:
    """Extract human-readable text from typed agent response."""
    output = agent_response.get("output", {})

    # Handle typed outputs
    if isinstance(output, dict):
        # ChatReply / QueryAnswer
        if "text" in output:
            return output["text"]
        if "answer" in output:
            return output["answer"]
        # ResearchReport
        if "findings" in output:
            findings = output["findings"]
            text = "\n".join(f"• {f['summary']}" for f in findings[:5])
            if output.get("next_questions"):
                text += "\n\nNext questions:\n" + "\n".join(
                    f"• {q}" for q in output["next_questions"][:3]
                )
            return text
        # CuratorResult / IngestResult — not user-facing in Telegram
        if "actions_taken" in output:
            return f"Done: {', '.join(output['actions_taken'][:3])}"
        if "note_path" in output:
            return f"Ingested → {output['note_path']}"

    # Fallback: stringify whatever we got
    return str(output)
```

### 2. Optionally surface citations in Telegram replies

For `ChatReply` and `QueryAnswer`, if `citations` are present, append a compact source list:

```python
def format_citations(citations: list[dict]) -> str:
    if not citations:
        return ""
    lines = [f"\n\n_Sources:_"]
    for c in citations[:3]:
        title = c.get("title", c.get("path", ""))
        lines.append(f"• {title}")
    return "\n".join(lines)
```
