# Plan 18-01: Restore VPS↔GitHub SSH auth (manual checkpoint)

## Goal

Restore authenticated git pull/push between the VPS `uabrain` user and `caiobellizzi/second-brain`.
The repo was made private, breaking the anonymous HTTPS remote — this is a prerequisite for all
subsequent Phase 18 work.

## Why type=checkpoint:manual

Part A requires interactive SSH access to the VPS as root, generating a keypair, registering it
on GitHub, and switching the git remote. The executor cannot perform these steps autonomously.
This plan pauses for operator action, then verifies completion.

<task type="checkpoint:manual" id="18-01-A-ssh-deploy-key">
  <title>Generate ed25519 deploy key and register on GitHub</title>

  <action>
    Run the following commands on the VPS (as root or uabrain):

    1. Check/create uabrain SSH dir:
       ```bash
       id uabrain
       sudo -u uabrain mkdir -p /home/uabrain/.ssh && sudo chmod 700 /home/uabrain/.ssh
       ```

    2. Generate keypair:
       ```bash
       sudo -u uabrain ssh-keygen -t ed25519 -f /home/uabrain/.ssh/second_brain_deploy -N ""
       ```

    3. Add SSH config for github.com (append to /home/uabrain/.ssh/config):
       ```
       Host github.com
         IdentityFile ~/.ssh/second_brain_deploy
         IdentitiesOnly yes
       ```
       Then: `sudo chmod 600 /home/uabrain/.ssh/config`

    4. Add github.com to known_hosts:
       ```bash
       sudo -u uabrain ssh-keyscan github.com >> /home/uabrain/.ssh/known_hosts
       ```

    5. Register public key as write-enabled deploy key (run from local machine):
       ```bash
       gh repo deploy-key add /tmp/second_brain_deploy.pub \
         --repo caiobellizzi/second-brain \
         --title vps-uabrain \
         --allow-write
       ```
       (scp the pubkey to local first: `scp root@31.97.130.253:/home/uabrain/.ssh/second_brain_deploy.pub /tmp/`)

    6. Switch VPS remote from HTTPS to SSH:
       ```bash
       git -C /srv/second-brain remote set-url origin git@github.com:caiobellizzi/second-brain.git
       ```
  </action>

  <verify>
    As uabrain on VPS:
    ```bash
    sudo -u uabrain git -C /srv/second-brain fetch origin main
    ```
    Must complete without "could not read Username" or authentication error.
    Also run a no-op push: `sudo -u uabrain /opt/ultra-agents-brain/scripts/git-sync.sh push`
    to confirm the write-enabled key works in both directions.
  </verify>

  <done>
    SSH deploy key is installed, VPS remote is SSH, both pull and push succeed as uabrain.
    Proceed to Plan 18-02.
  </done>
</task>
