  <name>Task 1: Write scripts/reindex_bridge.sh — post-commit hook for codebase-memory-mcp</name>
  <files>
    scripts/reindex_bridge.sh
  </files>
  <read_first>
    - agentos/workshop_registry.py (how repos are registered; how to get repo slug)
    - vault/CLAUDE.md §Path And Naming Rules (vault/repos/<repo>/ convention)
    - Check if codebase-memory-mcp CLI binary is on PATH: `which codebase-memory-mcp 2>/dev/null`
    - Check if the MCP has a reindex subcommand: `codebase-memory-mcp --help 2>/dev/null | head -20`
  </read_first>
  <action>
    Create `scripts/reindex_bridge.sh` as a portable bash script.

    Script logic:
    1. Detect the repo root (git rev-parse --show-toplevel).
    2. Extract the repo slug (basename of root dir, lowercased, kebab-cased).
    3. Set VAULT_PATH to ~/Documents/second-brain (or $VAULT_PATH env var if set).
    4. Run codebase-memory-mcp reindex (or `detect_changes` + re-index — check the CLI help first).
       If codebase-memory-mcp is not on PATH, log a warning and exit 0 (non-blocking).
    5. Export architecture summary: `codebase-memory-mcp get_architecture > /tmp/uab-arch-summary.txt`
       (use the correct CLI invocation — check help).
       If export fails, log warning and exit 0 (non-blocking — never block commits).
    6. Create vault/repos/<slug>/ if it does not exist.
    7. Write /tmp/uab-arch-summary.txt to vault/repos/<slug>/ARCHITECTURE.md with a header:
       # Architecture — <slug>
       Updated: <ISO timestamp>
       ---
       <content of summary>
    8. Exit 0 always — post-commit hooks must never block commits.

    Installation note (add as comment in script):
    # Install per repo: cp scripts/reindex_bridge.sh .git/hooks/post-commit && chmod +x .git/hooks/post-commit
  </action>
  <verify>
    <automated>
      bash scripts/reindex_bridge.sh 2>&1 | tail -10
    </automated>
    <human-check>
      Run the script from the ultra-agents-brain repo root.
      Check that vault/repos/ultra-agents-brain/ARCHITECTURE.md was created or updated.
      Verify the script exits 0 even if codebase-memory-mcp is not installed.
    </human-check>
  </verify>
  <acceptance_criteria>
    - scripts/reindex_bridge.sh exists and is executable.
    - Script exits 0 always (non-blocking).
    - When codebase-memory-mcp is on PATH and functional: vault/repos/<repo>/ARCHITECTURE.md is written.
    - Script handles missing codebase-memory-mcp gracefully with a warning log, no error exit.
  </acceptance_criteria>
  <done>
    Script runs; exits 0; ARCHITECTURE.md written when tool available; graceful fallback when not.
  </done>

    RED first: write tests/unit/test_spec_gen.py with these cases before implementing.

    Tests:
    1. `test_spec_gen_produces_all_required_fields` — call generate_spec() with a stub briefing
       (dict with title, context, concepts, sources) and a stub ARCHITECTURE.md string. Assert
       the returned SPEC.md string contains all required section headers: "Problem & Context",
       "In-Scope", "Out-of-Scope", "Interfaces", "Acceptance Criteria", "References",
       "Rails" (Always/Ask-First/Never), "Code Style Example".
    2. `test_spec_gen_embeds_architecture_snippet` — assert the ARCHITECTURE.md content appears
       (or is excerpted) in the Code Style Example section.
    3. `test_spec_gen_ears_format` — assert at least one acceptance criterion uses EARS form
       ("When ... the system shall ...") when stub briefing has a goal field.
    4. `test_spec_gen_references_source_wikilinks` — stub briefing has source_notes: ["[[ai-agents-overview]]"];
       assert output references section contains "[[ai-agents-overview]]".

    Tests must fail (RED) before implementation.

    GREEN: implement ultra_brain/spec_gen.py.

    Public interface:
    ```python
    def generate_spec(
        briefing: dict,          # vault briefing frontmatter + content
        architecture_md: str,    # content of vault/repos/<repo>/ARCHITECTURE.md
        repo_path: str = "",     # for reference links
    ) -> str:
        """Return SPEC.md content as a string."""
    ```

    Implementation:
    - Pure Python, no new dependencies. Uses string templating, not an LLM call.
      (LLM-assisted spec drafting is a future enhancement; Phase 16 delivers the structured template.)
    - Reads briefing dict fields: title, content/body, concepts (list), source_notes (list),
      telos_relevance, tags.
    - Builds SPEC.md sections from the briefing content + ARCHITECTURE.md snippet.
    - EARS criteria: if briefing content contains goal-like statements, template them into
      "When X, the system shall Y" format as placeholders for human completion.
    - Rails section: Always/Ask-First/Never as placeholder tables for human completion.
    - Code style example: first code block found in ARCHITECTURE.md, or placeholder.

    CLI entry point (for testing from command line):
    `python3 -m ultra_brain spec-gen --briefing <path> --repo <slug> --vault <vault_path>`

    Do NOT call agentos/knowledge.py in this phase — that requires the Postgres DB running.
    Keep spec_gen.py purely local (reads vault files directly). The PgVector integration
    is a future enhancement noted in comments.
