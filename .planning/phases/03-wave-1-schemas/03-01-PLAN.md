# Wave 1 — Schemas + Model Factory

## Objective

Create `agentos/schemas.py` with typed Pydantic result models for all 5 agents, and extend `agentos/model.py` with a `chat_model(tier)` helper. Both are prerequisites for Wave 2 (per-agent `output_schema=` wiring) and Wave 5 (eval field assertions).

## Context

v1.0 agents return untyped strings. The telegram adapter parses responses with brittle string extraction. Adding `output_schema=` to each agent forces structured JSON replies — typed contracts that the adapter, evals, and future workshop clients can rely on.

## Tasks

### 1. Create `agentos/schemas.py`

```python
from pydantic import BaseModel


class VaultCitation(BaseModel):
    path: str
    title: str
    tags: list[str] = []
    excerpt: str = ""


class ChatReply(BaseModel):
    text: str
    citations: list[VaultCitation] = []
    suggested_actions: list[str] = []


class QueryAnswer(BaseModel):
    answer: str
    citations: list[VaultCitation] = []
    confidence: float = 1.0


class IngestResult(BaseModel):
    note_path: str
    frontmatter: dict = {}
    tags: list[str] = []
    needs_review: bool = False


class Finding(BaseModel):
    summary: str
    source: str = ""


class ResearchReport(BaseModel):
    topic: str
    findings: list[Finding] = []
    next_questions: list[str] = []


class CuratorResult(BaseModel):
    actions_taken: list[str] = []
    notes_touched: list[str] = []
    errors: list[str] = []


class SupervisorRouting(BaseModel):
    chosen_member: str
    reason: str = ""
    response: str = ""
```

### 2. Extend `agentos/model.py` with `chat_model()` helper

Current `model.py` creates an `OpenAIChat` instance directly in each agent. Add a factory function that maps tier names to LiteLLM model IDs:

```python
def chat_model(tier: str = "default-worker") -> OpenAIChat:
    """Return an OpenAIChat model for the given tier.

    Tiers:
      cheap-worker    — fast/cheap local or cloud model
      default-worker  — balanced capability for most tasks
      orchestrator    — most capable model for planning/reasoning
      private-worker  — local LM Studio only (no cloud fallback)
    """
    tier_map = {
        "cheap-worker": os.getenv("LITELLM_CHEAP_MODEL", "cheap-worker"),
        "default-worker": os.getenv("LITELLM_DEFAULT_MODEL", "default-worker"),
        "orchestrator": os.getenv("LITELLM_ORCHESTRATOR_MODEL", "orchestrator"),
        "private-worker": os.getenv("LITELLM_PRIVATE_MODEL", "private-worker"),
    }
    model_id = tier_map.get(tier, tier_map["default-worker"])
    return OpenAIChat(
        id=model_id,
        base_url=os.getenv("LITELLM_BASE_URL", "http://localhost:4000/v1"),
        api_key=os.getenv("LITELLM_API_KEY", "sk-fake"),
    )
```

Keep existing direct instantiation in agents working — `chat_model()` is additive.
