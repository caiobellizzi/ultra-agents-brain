# Wave 2 Plan 4 — Query Agent Reconfiguration

## Objective

Upgrade `agentos/agents/query.py` to full feature set: MemoryManager, session summaries, agentic RAG, `ReasoningTools`, `QueryAnswer` output schema, model bump to `default-worker`, and inline `AgentAsJudgeEval` for citation quality.

Query is the agent that answers questions about vault content — it needs RAG more than any other agent.

## Context

v1.0 query.py is bare: model + `query_vault` tool, `cheap-worker` tier. This plan gives it the full conversational stack including session memory so it can remember what the user asked in previous sessions, vault RAG so it uses semantic search instead of exact-match tool calls, and reasoning so it can plan multi-step lookups.

### 3. Add kwargs to Agent instantiation

```python
EVAL_JUDGE_TIER = os.getenv("EVAL_JUDGE_TIER", "private-worker")

citation_judge = AgentAsJudgeEval(
    name="query-must-cite",
    criteria="The answer MUST cite at least one vault source. If no relevant vault content exists, say so explicitly rather than answering without citations.",
    scoring_strategy="binary",
    model=chat_model(EVAL_JUDGE_TIER),
    db=db,
    run_in_background=True,
)

agent = Agent(
    name="query",
    model=chat_model("default-worker"),  # bumped from cheap-worker
    # Memory + session summaries (conversational)
    memory_manager=memory_manager,
    enable_agentic_memory=True,
    update_memory_on_run=True,
    add_history_to_context=True,
    enable_session_summaries=True,
    add_session_summary_to_context=True,
    search_past_sessions=True,
    num_past_sessions_to_search=3,
    # Agentic RAG
    knowledge=knowledge,
    search_knowledge=True,
    # Typed output
    output_schema=QueryAnswer,
    # Reasoning + tools
    tools=[ReasoningTools(add_instructions=True), query_vault],
    # Inline judge
    evals=[citation_judge],
    storage=db,
)
```
