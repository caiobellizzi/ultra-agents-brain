# Wave 2 Plan 2 — Curator Agent Reconfiguration

## Objective

Upgrade `agentos/agents/curator.py` with shared MemoryManager and typed `CuratorResult` output schema. Curator is a bulk/one-shot agent — no session summaries, no RAG.

## Context

Curator runs scheduled bulk operations (`run_digest`, `run_review`, `lint_vault`, `poll_feeds`). It doesn't need cross-session memory summaries or vault RAG (it writes TO the vault, not reads from it). Only the shared memory stack and typed output are needed.

Model tier stays `cheap-worker` — bulk ops don't need orchestrator capability.

## Tasks

### 1. Update imports in `agentos/agents/curator.py`

Add:
```python
from agno.memory.manager import MemoryManager
from agentos.schemas import CuratorResult
```

### 2. Update `make_curator_agent()` function signature

```python
def make_curator_agent(
    memory_manager: MemoryManager,
    db,
) -> Agent:
```

### 3. Add kwargs to Agent instantiation

```python
agent = Agent(
    name="curator",
    model=chat_model("cheap-worker"),
    # Memory (no session summaries — one-shot bulk)
    memory_manager=memory_manager,
    enable_agentic_memory=True,
    update_memory_on_run=True,
    add_history_to_context=True,
    # Typed output
    output_schema=CuratorResult,
    # Existing tools preserved
    tools=[run_digest, run_review, lint_vault, poll_feeds],
    storage=db,
)
```

No `knowledge=`, no `enable_session_summaries=`, no `ReasoningTools`.

### 4. Update tests

Extend curator fixture in `tests/test_agentos.py` to mock `memory_manager`. Assert `output_schema=CuratorResult`.
