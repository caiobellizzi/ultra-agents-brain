# Wave 2 Plan 1 — Chat Agent Reconfiguration

## Objective

Upgrade `agentos/agents/chat.py` to the full Agno 2.6.7 feature set: shared MemoryManager, session summaries, agentic RAG over vault, and typed `ChatReply` output schema.

Chat is the primary user-facing agent — configure it first so it can be tested via Telegram before committing the other 4 agents.

## Context

Current chat.py already has `enable_agentic_memory` and session summaries (partially configured from v1.0). This plan replaces the inline `db` references with the shared `MemoryManager` from `agentos/app.py`, adds `knowledge=` for vault RAG, and adds `output_schema=ChatReply`.

The `memory` and `knowledge` objects are instantiated in `agentos/app.py` (Wave 3) — chat.py will receive them as constructor arguments. For this wave, update the agent factory function signature to accept them.

### 4. Inline AgentAsJudge eval (citation judge)

```python

import os

EVAL_JUDGE_TIER = os.getenv("EVAL_JUDGE_TIER", "private-worker")

citation_judge = AgentAsJudgeEval(
    name="chat-must-cite",
    criteria="If the user's message references something likely in their vault (notes, articles, ideas), the response MUST include at least one vault citation. If purely conversational or unrelated to vault content, pass.",
    scoring_strategy="binary",
    model=chat_model(EVAL_JUDGE_TIER),
    db=db,
    run_in_background=True,
```

Add `evals=[citation_judge]` to the Agent instantiation.
