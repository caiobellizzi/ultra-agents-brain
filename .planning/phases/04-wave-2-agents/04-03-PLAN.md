# Wave 2 Plan 3 — Ingest Agent Reconfiguration

## Objective

Upgrade `agentos/agents/ingest.py` with MemoryManager, `ReasoningTools`, `IngestResult` output schema, and model bump to `default-worker`. Ingest needs reasoning to correctly categorize notes and structured output so callers get typed `IngestResult` (path, tags, needs_review flag).

## Context

Ingest has HITL confirmation (`@tool(requires_confirmation=True)` on `ingest_to_vault`). That flow must be preserved unchanged. The `output_schema=IngestResult` will wrap the final response; the HITL tool call itself is unaffected.

Model bumps from `cheap-worker` to `default-worker` — structured output generation requires a model capable of reliably following Pydantic schemas.

    name="ingest",
    model=chat_model("default-worker"),  # bumped from cheap-worker
    # Memory (no session summaries — one-shot)

    enable_agentic_memory=True,
