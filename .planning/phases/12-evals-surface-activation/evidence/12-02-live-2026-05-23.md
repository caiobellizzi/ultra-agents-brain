The end-to-end behaviour the local test was designed to exercise was executed instead via an inline `python - << EOF` heredoc that runs the same `EvalRunRecord` write → `SELECT` → `DELETE` roundtrip against the live `ai.agno_eval_runs` table. ## Command

```bash
ssh root@31.97.130.253 'cd /opt/ultra-agents-brain && set -a && . ./.env && set +a && .venv/bin/python - << PYEOF
import os, uuid
from sqlalchemy import create_engine, text
from agno.db.postgres import PostgresDb
from agno.db.schemas.evals import EvalRunRecord, EvalType

dsn = os.environ["POSTGRES_DSN_SESSIONS"]
db = PostgresDb(id="live-eval-suite", db_url=dsn, create_schema=True)
run_id = f"int-{uuid.uuid4()}"

record = EvalRunRecord(
    run_id=run_id,
    eval_type=EvalType.ACCURACY,
    agent_id="integration-test",
    model_id="test/model",
    model_provider="test",
    eval_input={"user_message": "test input"},
    eval_data={
        "score": 0.5, "output": "test output", "latency_ms": 42,
        "model_id": "test/model", "model_provider": "test", "status": "ok",
    },
)
db.create_eval_run(record)

engine = create_engine(dsn)
with engine.begin() as conn:
    row = conn.execute(
        text("SELECT eval_type, run_id, agent_id FROM ai.agno_eval_runs WHERE run_id = :rid"),
        {"rid": run_id},
    ).first()
    assert row is not None and row.eval_type == "accuracy" and row.agent_id == "integration-test"
    print(f"OK: row found — run_id={row.run_id}, eval_type={row.eval_type}, agent_id={row.agent_id}")
    deleted = conn.execute(
        text("DELETE FROM ai.agno_eval_runs WHERE run_id = :rid RETURNING run_id"),
        {"rid": run_id},
    ).fetchall()
    print(f"OK: cleaned up {len(deleted)} row(s)")
engine.dispose()
print("LIVE_TEST_PASSED")
PYEOF'
```

- The `ai.agno_eval_runs` table accepts a row with `eval_type='accuracy'` from a Python `EvalRunRecord` constructed with `EvalType.ACCURACY` — D-07 (use enum, not string) is correct under live conditions.
- The cleanup `DELETE` succeeded — the test row does not persist in production.
