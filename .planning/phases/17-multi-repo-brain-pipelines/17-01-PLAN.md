# Phase 17-01: Multi-Repo Brain Pipelines

## Goal

Make all enrolled repos queryable from Telegram/anywhere by compiling nightly LLM prose summaries via GitHub Actions and landing them in the second-brain vault as `repos/<name>.md` files.

The codebase-memory-mcp graph is Mac-local — it cannot serve the VPS Telegram bot or teammates. This phase replaces the current useless `reindex_bridge.sh` vault-write (raw stats JSON) with a structured LLM prose summary per repo, compiled in CI.

## Architecture

```
enrolled repo (nightly cron)
  └── .github/workflows/summary.yml  (5-line caller stub)
        └── brain-pipelines/generate-summary.yml  (@v1)
              ├── checkout repo
              ├── collect: README + .planning/ + git log (30d) + file tree
              └── NVIDIA NIM → SUMMARY.md artifact

second-brain repo (staggered ~2h later)
  └── aggregate.yml
        ├── download all SUMMARY.md artifacts
        └── write repos/<name>.md per repo → push to main

VPS + Mac
  └── git -C /srv/second-brain pull  (cron)
        └── Telegram BM25 search picks up repos/*.md automatically
```

## Tasks

### Task 1 — Create brain-pipelines repo and reusable workflow

Create new public GitHub repo `brain-pipelines` with:
- `generate-summary.yml` reusable workflow (workflow_call trigger)
- Input: repo name, checkout token
- Collects: README.md, .planning/ directory, `git log --oneline --since=30d`, top-level file tree
- Calls NVIDIA NIM API (`meta/llama-3.1-70b-instruct` or equivalent) with a structured prompt
- Output schema for `repos/<name>.md`: Purpose, Stack, Architecture, Status, Recent activity, Key decisions, Entry points, Links
- Uploads SUMMARY.md as a GitHub Actions artifact
- Tag floating `@v1` on first stable commit

**Verify:** Manually trigger workflow on `ultra-agents-brain`; artifact appears with valid markdown.

### Task 2 — Create aggregate.yml in second-brain

Add `.github/workflows/aggregate.yml` to `second-brain` repo:
- Runs on schedule: `0 4 * * *` (4am UTC, ~2h after enrolled repos at 2am)
- Downloads SUMMARY.md artifacts from all enrolled repos via `gh run download`
- Writes each to `repos/<name>.md` and commits to main
- Uses `GITHUB_TOKEN` (no extra secret needed for same-org artifact download)

**Verify:** After nightly run, `git log second-brain` shows a new commit with `repos/ultra-agents-brain.md`.

### Task 3 — Add caller stub to enrolled repos

For each enrolled repo (starting with `ultra-agents-brain`), add:

```yaml
# .github/workflows/summary.yml
name: Brain Summary
on:
  schedule: [{cron: "0 2 * * *"}]
  workflow_dispatch:
jobs:
  summarize:
    uses: <owner>/brain-pipelines/.github/workflows/generate-summary.yml@v1
    secrets:
      NVIDIA_NIM_API_KEY: ${{ secrets.NVIDIA_NIM_API_KEY }}
```

**Verify:** Workflow appears in Actions tab; manual trigger succeeds.

### Task 4 — Wire NVIDIA NIM secret

Add `NVIDIA_NIM_API_KEY` as a GitHub Actions secret to:
- Each enrolled repo (or org-level secret if multiple repos share the org)
- The `brain-pipelines` repo (for testing)

**Verify:** `generate-summary.yml` runs without auth errors.

### Task 5 — Remove vault-write block from reindex_bridge.sh

In `reindex_bridge.sh`, remove the block that writes `vault/repos/ARCHITECTURE.md` (raw stats JSON). Preserve all `codebase-memory-mcp` reindex calls.

**Verify:** `bash reindex_bridge.sh` completes without writing to `vault/repos/`; `~/.cache/codebase-memory-mcp/` DB is still updated.

### Task 6 — Fix ops/sync-vault-to-vps.sh to exclude repos/

Add `--exclude 'repos/'` to the rsync call in `ops/sync-vault-to-vps.sh` so git-managed `repos/*.md` files are not overwritten or deleted by rsync.

**Verify:** Run sync script; `repos/ultra-agents-brain.md` on VPS survives the sync.

### Task 7 — Smoke verification

After tasks 1–6 complete:
1. Manually trigger `.github/workflows/summary.yml` in `ultra-agents-brain`
2. Wait for `aggregate.yml` in `second-brain` (or trigger manually)
3. Confirm `repos/ultra-agents-brain.md` appears in `second-brain` with correct schema
4. Confirm Telegram `/search ultra-agents-brain` returns a result
5. Confirm VPS `second-brain` has the file after `git pull`

## Anti-Scope

- No vector RAG / pgvector — BM25 search is sufficient for ~10–20 repos at personal scale
- No per-commit summaries — nightly cadence only
- No forced enrollment — opt-in per repo
- No reindex_bridge.sh removal of codebase-memory-mcp reindex calls (local graph preserved for Claude Code)
- No changes to the Telegram bot search logic (it already picks up new files in the vault)

## Version Propagation

- Floating tag `@v1` in `brain-pipelines` is force-pushed for patch updates
- Breaking changes (v1→v2) managed via Renovate auto-PRs in enrolled repos
