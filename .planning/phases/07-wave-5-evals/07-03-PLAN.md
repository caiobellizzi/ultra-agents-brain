# Wave 5 Plan 3 — Baselines + Pre-commit Hook (Commit 5c)

## Objective

Run evals once with cloud-tier judge to freeze baseline scores, commit the baseline JSONs, create the pre-commit router script, and wire it in `.pre-commit-config.yaml`. After this commit, committing agent files triggers scoped eval runs as a quality gate.

## Context

Pre-commit routing replaces scheduled CI/systemd eval runs entirely. Benefits: developer feedback in ≤15s for single-agent edits, no always-on infra, eval runs only when relevant files change.

Per-file mapping (locked in grill session):

| Files staged | Evals run |
|---|---|
| `agentos/agents/chat.py` | `pytest evals/test_chat.py` |
| `agentos/agents/curator.py` | `pytest evals/test_curator.py` |
| `agentos/agents/ingest.py` | `pytest evals/test_ingest.py` |
| `agentos/agents/query.py` | `pytest evals/test_query.py` |
| `agentos/agents/research.py` | `pytest evals/test_research.py` |
| `agentos/agents/supervisor.py` | `pytest evals/test_supervisor.py` |
| `agentos/knowledge.py` | `pytest evals/test_query.py evals/test_research.py` |
| `agentos/model.py`, `agentos/app.py`, `agentos/schemas.py` | `pytest evals/` (full) |
| Any other file | (skip evals) |

## Tasks

### 1. Add `--write-baseline` and `--update-baseline` flags to `evals/conftest.py`

```python
def pytest_addoption(parser):
    parser.addoption("--write-baseline", action="store_true", default=False)
    parser.addoption("--update-baseline", action="store_true", default=False)

@pytest.fixture(scope="session")
def write_baseline(request):
    return request.config.getoption("--write-baseline")
```

In accuracy/performance tests, when `write_baseline=True`, write results to `evals/baselines/*.json` instead of asserting against them.

### 2. Run baseline generation

```bash
EVAL_JUDGE_TIER=orchestrator pytest evals/ --write-baseline
```

This creates `evals/baselines/accuracy_baseline.json` and `evals/baselines/performance_baseline.json`. Commit these files.

Baseline format:
```json
{
  "chat-accuracy-case-1": {"avg_score": 0.85, "generated_at": "2026-05-20"},
  "chat-accuracy-case-2": {"avg_score": 0.90, "generated_at": "2026-05-20"}
}
```

### 3. Create `tools/precommit_eval_router.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

STAGED=$(git diff --cached --name-only)

EVALS_TO_RUN=""

for file in $STAGED; do
  case "$file" in
    agentos/agents/chat.py)      EVALS_TO_RUN="$EVALS_TO_RUN evals/test_chat.py" ;;
    agentos/agents/curator.py)   EVALS_TO_RUN="$EVALS_TO_RUN evals/test_curator.py" ;;
    agentos/agents/ingest.py)    EVALS_TO_RUN="$EVALS_TO_RUN evals/test_ingest.py" ;;
    agentos/agents/query.py)     EVALS_TO_RUN="$EVALS_TO_RUN evals/test_query.py" ;;
    agentos/agents/research.py)  EVALS_TO_RUN="$EVALS_TO_RUN evals/test_research.py" ;;
    agentos/agents/supervisor.py) EVALS_TO_RUN="$EVALS_TO_RUN evals/test_supervisor.py" ;;
    agentos/knowledge.py)        EVALS_TO_RUN="$EVALS_TO_RUN evals/test_query.py evals/test_research.py" ;;
    agentos/model.py|agentos/app.py|agentos/schemas.py) EVALS_TO_RUN="evals/" ;;
  esac
done

# Deduplicate
EVALS_TO_RUN=$(echo "$EVALS_TO_RUN" | tr ' ' '\n' | sort -u | tr '\n' ' ')

if [ -z "$EVALS_TO_RUN" ]; then
  exit 0  # No relevant files staged
fi

echo "Running scoped evals: $EVALS_TO_RUN"
exec pytest $EVALS_TO_RUN --tb=short -q
```

```bash
chmod +x tools/precommit_eval_router.sh
```

### 4. Create `.pre-commit-config.yaml`

```yaml
repos:
  - repo: local
    hooks:
      - id: scoped-evals
        name: Scoped agent evals
        entry: tools/precommit_eval_router.sh
        language: script
        pass_filenames: false
        always_run: false
        stages: [commit]
```

If `.pre-commit-config.yaml` already exists, add the `scoped-evals` hook entry to the `repos:` list.

### 5. Install pre-commit hook

```bash
pip install pre-commit
pre-commit install
```

## Verification

| Check | Command |
|---|---|
| Baselines frozen | `ls evals/baselines/` shows both JSON files |
| Single-agent commit fast | Stage only `agentos/agents/chat.py` → commit → only `test_chat.py` runs (≤15s) |
| Full-suite trigger works | Stage `agentos/model.py` → commit → full `pytest evals/` runs |
| Unrelated files skip | Stage `channels/telegram_adapter.py` → commit → evals skipped |
| Regression gate works | Lower accuracy_baseline.json score → stage chat.py → commit blocked; restore → succeeds |
| Judge tier switch | `EVAL_JUDGE_TIER=orchestrator pytest evals/test_chat.py` produces different scores than default |
