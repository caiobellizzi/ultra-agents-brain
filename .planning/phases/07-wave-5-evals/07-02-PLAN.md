# Wave 5 Plan 2 — Eval Coverage + Inline Judges (Commit 5b)

## Objective

Fill the 18 hand-authored test cases across 6 datasets, implement all eval types per the coverage matrix, and attach inline `AgentAsJudgeEval` to production agent files (chat, query, research). After this commit, `pytest evals/` runs the full test suite with real agent calls.

**Authoring cost: ~3 hours.** `autonomous: false` — cases require domain knowledge of the vault content and expected agent behavior.

## Context

Eval strategy is output-type-driven (locked in grill session):
- **Pydantic fields** → plain pytest assertions (deterministic, fast)
- **Free-text chat reply** → `AccuracyEval` with hand-authored expected text
- **Tool selection** → `ReliabilityEval`
- **Latency baselines** → `PerformanceEval`
- **Continuous quality** → inline `AgentAsJudgeEval` in production agent files (already added in Wave 2 plans)

**Critical gotchas:**
- Default judge is `o4-mini` — always pass explicit `model=judge_model` fixture
- `PerformanceEval.func` must be sync — wrap async calls in `asyncio.run()`
- `AccuracyEval.run()` raises `ValueError` if `db` is async — we're sync, fine

### 2. Fill dataset cases (3 per agent)

Each dataset follows the pattern:
```python
CASES = [
    {
        "id": "case-1",
        "input": "<real user query or task>",
        # Field assertions (for Pydantic outputs)
        "expected_citations_have_tag": "#eval-seed",  # or None
        "expected_min_field": {"citations": 1},       # e.g. len(citations) >= 1
        # For AccuracyEval (chat only)
        "expected_text": "...",  # hand-authored expected reply
        # For ReliabilityEval
        "expected_tool_calls": ["ingest_to_vault"],
        # For PerformanceEval
        "max_latency_seconds": 30,
    },
]
```

**chat_cases.py** — 3 cases covering: vault-referenced question (expect citation), conversational (expect no citation), multi-turn context recall.

**query_cases.py** — 3 cases: direct vault query (expect #eval-seed citation), compound question, no-vault-content question (expect explicit "no vault content" response).

**ingest_cases.py** — 3 cases: article URL ingest (expect `note_path` starts with `vault/02-Resources/`), personal note (expect `tags >= 2`), ambiguous content (expect `needs_review=True`).

**research_cases.py** — 3 cases: topic with vault background (expect `next_questions >= 3`), novel topic (expect `findings non-empty`), follow-up (expect reasoning tool calls present in trace).

**curator_cases.py** — 3 cases: lint run (expect `actions_taken non-empty, errors empty`), digest run (expect `notes_touched >= 1`), review run (expect no errors).

**supervisor_routing.py** — 3 cases: vault question → expect `chosen_member=query`, research topic → expect `chosen_member=research`, ingest request → expect `chosen_member=ingest`.

### 3. Implement test_chat.py

```python
from agno.eval.accuracy import AccuracyEval
from agno.eval.performance import PerformanceEval
from evals.datasets.chat_cases import CHAT_CASES

@pytest.mark.parametrize("case", CHAT_CASES)
def test_chat_field_assertions(case, chat_agent):
    result = chat_agent.run(case["input"])
    reply = result.output
    assert reply.text  # non-empty
    if case.get("expected_citations_have_tag"):
        citation_tags = {tag for c in reply.citations for tag in c.tags}
        assert case["expected_citations_have_tag"] in citation_tags

def test_chat_accuracy(judge_model, eval_db):
    accuracy_case = CHAT_CASES[0]  # the vault-referenced case
    eval = AccuracyEval(
        name="chat-accuracy",
        expected=accuracy_case["expected_text"],
        model=judge_model,
        db=eval_db,
    )
    # ... run and assert avg_score >= 0.7

def test_chat_performance(eval_db):
    eval = PerformanceEval(
        name="chat-perf",
        func=lambda: chat_agent.run(CHAT_CASES[0]["input"]),
        db=eval_db,
    )
    result = eval.run(num_iterations=3)
    assert result.avg_run_time < 30
```

Repeat the pattern for each agent's test file per the coverage matrix.
