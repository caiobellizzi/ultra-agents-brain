# Phase 1 — Plan 1: Build ultra-brain on Agno

## Context

Replaces the hallucinated Hermes Docker image with a real, production-grade agent runtime.
The existing `ultra_brain/*.py` modules are WRAPPED (not rewritten) as Agno tools. A new
`agentos/` FastAPI package hosts all agents; `channels/telegram_adapter.py` becomes a dumb
adapter that long-polls Telegram and POSTs to AgentOS. Systemd services replace the old
`ultra-agents-brain.service`.

Architecture decision: Brain-first. `ultra-workshop` (coding agent team) is deferred until
this Brain is running and used daily for 2–4 weeks.

### Task 2.6 — AgentOS app

**Files:** `agentos/__init__.py`, `agentos/app.py` (~25 LOC), `agentos/__main__.py` (~15 LOC), `agentos/db.py`, `agentos/model.py`

**Implementation update (2026-05-19):** Uses Agno's built-in `agno.os.app.AgentOS` class (not a hand-rolled FastAPI app). This exposes the standard AgentOS HTTP schema and enables the hosted dashboard at https://os.agno.com to connect.

Routes exposed by Agno (not custom):
- `POST /agents/{agent_id}/runs` — run an agent (chat, ingest, query, research, curator)
- `POST /agents/{agent_id}/runs/{run_id}/{continue|cancel|resume}` — HITL flow
- `GET /agents`, `GET /agents/{agent_id}` — agent introspection
- `GET /approvals`, `POST /approvals/{id}/resolve` — trust-gate approvals
- `GET /knowledge/search`, `/knowledge/content` — vault retrieval
- `GET /sessions`, `/memories`, `/metrics`, `/health`, `/docs`

**Request shape:** form-data with `message=...`, `session_id=...`, `stream=true|false` (not JSON body).

**Verify:** `python -m agentos` starts on :7000; `curl -X POST :7000/agents/chat/runs -F "message=hi" -F "stream=false"` returns `{"content":"...","run_id":"...","session_id":"..."}`. Dashboard at https://os.agno.com → Add OS → http://localhost:7000 lists all 5 agents.

## Wave 3 — Telegram adapter

**Goal:** End-to-end Telegram flow works locally before VPS deploy.

### Task 3.1 — Build adapter

**Files:** `channels/__init__.py`, `channels/telegram_adapter.py` (~140 LOC)

Long-poll Telegram → POST to AgentOS → reply. Routing:

| Telegram command | AgentOS call |
|---|---|
| plain text | `POST /agents/chat/runs` |
| `/ingest <url>` | `POST /agents/ingest/runs` |
| `/query <text>` | `POST /agents/query/runs` |
| `/research <topic>` | `POST /agents/research/runs` |

Request body is **form-data** (not JSON): `message=<text>&session_id=<chat_id>&user_id=<tg_user>&stream=false`.

HITL flow (medium-risk actions):
1. Initial run returns `status: "PAUSED"` with an `approval_id` in the response payload.
2. Adapter renders Telegram inline buttons "✅ Approve" / "❌ Deny" with `callback_data=approve:{approval_id}` / `deny:{approval_id}`.
3. On user click, adapter calls `POST /approvals/{approval_id}/resolve` with `{"approved": true|false}` (JSON body).
4. Adapter then calls `POST /agents/{id}/runs/{run_id}/continue` to resume the paused run.
5. Final reply (note path or refusal) is sent back to the Telegram chat.

Session key strategy: `session_id = f"telegram-{chat_id}"` so each Telegram chat has its own Agno session memory.

### Task 3.2 — Local smoke test

Run `python -m channels.telegram_adapter` on Mac. Send "hi" to `@ultra_agents_brain_bot`. Expect reply.

### Task 3.3 — Approval flow test

## Risk flags

- Agno API surface evolves — pinned `agno==2.6.7`; don't auto-upgrade. Dashboard at os.agno.com may move faster than the SDK; the open AG-UI protocol is the local-only fallback.
- Telegram adapter (Wave 3) depends on Agno's `POST /agents/{id}/runs` returning a stable shape including `run_id`, `session_id`, `content`, and `status: "PAUSED"` on HITL — verified locally during Wave 2 smoke. Re-verify after any agno upgrade.
- Approval flow round-trip (`/approvals/{id}/resolve` → `/runs/{run_id}/continue`) is untested with our trust_gate decorator. First Wave 3 task: confirm `HumanInLoopRequired` raised by `trust_gate` surfaces as `status: "PAUSED"` with an approval record.
- Bot token needs rotation via BotFather `/revoke` after Phase 5 passes
- `/srv/second-brain` may be near-empty initially — curator will have little to digest for first days
