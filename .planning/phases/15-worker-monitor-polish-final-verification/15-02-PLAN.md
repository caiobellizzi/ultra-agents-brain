# Plan 15-02 — Vault sync regression tests (MON-02)

## Objective

Add regression tests that prove `ops/sync-vault-to-vps.sh`'s 2-pass strategy protects VPS-generated Inbox items from being deleted by the `--delete` flag. The script itself is already correct — the deliverable is the test coverage.

## Background

The script currently:
- Pass 1: `rsync -av --update` (VPS → Mac, no `--delete`) — VPS-generated items land on Mac
- Pass 2: `rsync -av --update --delete` (Mac → VPS) — items now exist on Mac, so not deleted

The fix was already applied in a prior session. This plan closes MON-02 by adding tests that would catch any future regression (e.g., someone removing pass 1 or swapping pass order).

```python
def test_pull_before_push_delete():
    script = Path("ops/sync-vault-to-vps.sh").read_text()
    rsync_lines = [l.strip() for l in script.splitlines()
                   if l.strip().startswith("/usr/bin/rsync") or l.strip().startswith("rsync")]
    assert len(rsync_lines) >= 2, "Expected at least 2 rsync invocations"
    pull_lines = [l for l in rsync_lines if "--delete" not in l]
    push_lines = [l for l in rsync_lines if "--delete" in l]
    assert pull_lines, "No pull pass found (rsync without --delete)"
    assert push_lines, "No push pass found (rsync with --delete)"
    pull_idx = rsync_lines.index(pull_lines[0])
    push_idx = rsync_lines.index(push_lines[0])
    assert pull_idx < push_idx, "Pull (no --delete) must precede push (--delete)"
```

**Test 2 — Functional rsync simulation:**

```python
import subprocess, shutil, unittest
from pathlib import Path

class TestSyncDeleteSafety(unittest.TestCase):
    def test_vps_generated_items_survive_delete_sync(self):
        # Skip if rsync not available (CI without rsync)
        if shutil.which("rsync") is None:
            self.skipTest("rsync not available")

        with tempfile.TemporaryDirectory() as tmp:
            local = Path(tmp) / "local" / ""   # trailing slash = rsync dir content
            remote = Path(tmp) / "remote" / ""
            local.mkdir(); remote.mkdir()

            # Simulate a VPS-generated Inbox item (exists on remote only)
            (remote / "Inbox").mkdir()
            vps_item = remote / "Inbox" / "2026-05-27-vps-generated.md"
            vps_item.write_text("# VPS item\n")

            # Simulate a Mac-side file that exists on local only
            (local / "00-Projects").mkdir()
            local_item = local / "00-Projects" / "project.md"
            local_item.write_text("# Local project\n")

            # Pass 1: remote → local (no --delete)
            subprocess.run(
                ["rsync", "-a", "--update", f"{remote}/", f"{local}/"],
                check=True
            )

            # Pass 2: local → remote (--delete)
            subprocess.run(
                ["rsync", "-a", "--update", "--delete", f"{local}/", f"{remote}/"],
                check=True
            )

            # VPS-generated item must still exist on remote
            self.assertTrue(
                (remote / "Inbox" / "2026-05-27-vps-generated.md").exists(),
                "VPS-generated Inbox item was deleted by --delete sync"
            )
            # Local-only file must also exist on remote after push
            self.assertTrue(
                (remote / "00-Projects" / "project.md").exists(),
                "Mac-side file was not pushed to remote"
            )
```
