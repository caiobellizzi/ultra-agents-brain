# Wave 0 — Postgres + pgvector Infrastructure

## Objective

Stand up Postgres 16 + pgvector on the VPS as a new systemd service and add the four missing Python dependencies. This is the foundation for all subsequent waves — agents cannot use semantic memory, agentic RAG, or eval persistence without it.

## Context

v1.0 uses SQLite (`aiosqlite`) for session storage via `agno`. SQLite handles the current single-user load but cannot support PgVector (required for vault RAG), `PostgresDb` (required for Agno's full memory stack), or eval result persistence. This wave replaces it.

The VPS already runs 4 systemd units (`uab-agentos`, `uab-litellm`, `uab-telegram`, `uab-cron`). Pattern: all units in `ops/systemd/`, bound to loopback only, `restart=on-failure`.

## Tasks

### 1. Create `ops/systemd/uab-postgres.service`

```ini
[Unit]
Description=Postgres 16 + pgvector for ultra-agents-brain
After=network.target

[Service]
Type=forking
User=postgres
ExecStart=/usr/lib/postgresql/16/bin/pg_ctl start -D /var/lib/postgresql/16/main -l /var/log/postgresql/uab-postgres.log
ExecStop=/usr/lib/postgresql/16/bin/pg_ctl stop -D /var/lib/postgresql/16/main
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

Note: If using apt-installed postgres, the service may already exist as `postgresql.service`. In that case, this unit is a wrapper/alias or simply document that the existing service is used and add the `uab-postgres.service` as a drop-in that ensures the extension is loaded.

### 2. Install Postgres 16 + pgvector on VPS

Manual VPS steps (document in `ops/README.md`):
```bash
sudo apt install -y postgresql-16 postgresql-16-pgvector
sudo systemctl enable --now postgresql
```

### 3. Create databases and install extension

```sql
-- Run as postgres superuser
CREATE DATABASE agno_sessions;
CREATE DATABASE agno_knowledge;

\c agno_sessions
CREATE EXTENSION IF NOT EXISTS vector;

\c agno_knowledge
CREATE EXTENSION IF NOT EXISTS vector;
```

### 4. Create app user and grant permissions

```sql
CREATE USER uab WITH PASSWORD '<from .env>';
GRANT ALL ON DATABASE agno_sessions TO uab;
GRANT ALL ON DATABASE agno_knowledge TO uab;
```

### 5. Update `requirements.txt`

Add to requirements.txt:
```
psycopg[binary]>=3.2
pgvector>=0.3
sentence-transformers>=3.0
a2a-sdk>=0.2
```

### 6. Update `.env.example`

Add:
```
POSTGRES_DSN_SESSIONS=postgresql+psycopg://uab:<password>@127.0.0.1:5432/agno_sessions
POSTGRES_DSN_KNOWLEDGE=postgresql+psycopg://uab:<password>@127.0.0.1:5432/agno_knowledge
```

### 7. Smoke-test the connection locally

```bash
psql "$POSTGRES_DSN_SESSIONS" -c 'SELECT 1'
psql "$POSTGRES_DSN_KNOWLEDGE" -c "SELECT extname FROM pg_extension WHERE extname='vector'"
```
