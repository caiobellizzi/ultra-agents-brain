## Goal

Upgrade all 5 agents from minimal v1.0 config to Agno 2.6.7 production-grade feature set:
Postgres/pgvector, per-user semantic memory, agentic RAG over vault, ReasoningTools,
Pydantic-typed outputs, MCP server, A2A protocol, comprehensive evals, and multi-tier
model routing via LiteLLM (NIM + cloud + local fallbacks).

## Key Accomplishments

1. **Postgres-backed AgentOS** — single DB (sessions + knowledge + evals); SqliteDb fallback for dev/test.
2. **5 fully reconfigured agents** — every agent now ships with semantic memory, RAG, ReasoningTools, and Pydantic output schemas.
3. **MCP + A2A exposed** — `enable_mcp_server=True` on AgentOS, `a2a-sdk` pinned <1.0 for agno 2.6.7 compatibility, discoverable by future ultra-workshop.
4. **48-case eval suite + pre-commit router** — ≤15s single-agent, ≤90s full suite; `EVAL_JUDGE_TIER` env-var lets release builds swap to a stronger judge.
5. **5-tier LiteLLM matrix with NIM** — NVIDIA NIM joins Groq/Anthropic as a third cloud backend; research agent gets a dedicated `research-worker` tier; capability-routed fallbacks so tools + response_format don't fight each other.
6. **LiteLLM strip-response-format hook** — pre-call hook drops `response_format` for local models that combine it with tools (fixes Groq HTTP 400).
