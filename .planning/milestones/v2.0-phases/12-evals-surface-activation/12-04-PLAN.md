# Plan 12-04: Eval Row Semantics and Live Judge Worker

Imported from: `plans/eval-fixes-live-judge-plan.md`

## Objective

Correct Phase 12 eval row semantics after live verification showed unjudged production runs appearing in Agno as `Untitled Evaluation` rows with `eval_type=agent_as_judge` and `score=N/A`.

Keep the high-level Phase 12 requirements:

- EVAL-01: live agent runs create dashboard-visible eval rows.
- EVAL-02: the eval suite writes scored per-case rows.
- EVAL-03: judge tier swaps remain observable.
- OBS-01: eval writes emit structured logs.

Update the implementation decisions:

- unjudged live telemetry rows are `performance`
- scored suite cases remain `accuracy`
- judged live rows are child `agent_as_judge` rows
- async live judging is an opt-in worker, disabled by default

## Decision Update

This plan supersedes two Phase 12 implementation decisions without invalidating Phase 12's requirement set.

- Old D-07: live rows use `EvalType.AGENT_AS_JUDGE`.
- New D-16: live, unjudged rows use `EvalType.PERFORMANCE` because they record what happened, not how good it was.

- Old D-08: async background scoring is out of scope.
- New D-17: async background scoring is in scope for this corrective follow-up, provided it is disabled by default, sampled, privacy-gated, and non-blocking.

Still valid from Phase 12:

- D-01/D-02: post-run recorder applied to all Agent/Team paths.
- D-03/D-04/D-05/D-06: payload capture, log-and-swallow, DB id from instance, score remains null for unjudged parent rows.
- D-09/D-10/D-11/D-12: suite hook, conditional DB writes, baseline files remain, judge tier smoke remains.
- D-14/D-15: OBS-01 eval logging remains mandatory.

## Row Semantics

Live performance rows:

- `eval_type`: `performance`
- `name`: `live:<agent_id>`
- `evaluated_component_name`: `<agent_id>`
- `agent_id`: `<agent_id>`
- `eval_data.score`: `None`
- `eval_data.status`: `ok` or `error`
- `eval_data.latency_ms`: live run latency
- `eval_data.model_id`: live agent model id
- `eval_data.model_provider`: live agent model provider

Suite accuracy rows:

- `eval_type`: `accuracy`
- `name`: `eval-suite:<agent_id>:<case_id>`
- `evaluated_component_name`: `<agent_id>`
- deterministic `run_id`: `suite:<agent_id>:<case_id>:<git_identity>`

`git_identity` is based on `HEAD` plus a dirty hash when staged, unstaged, or untracked non-ignored files under `agentos/`, `evals/`, `tests/`, `ultra_brain/`, or `skills/` are modified. Include judge tier/model in the identity only when a real LLM judge determines the score.

Live judged rows:

- `eval_type`: `agent_as_judge`
- `name`: `eval-live:<agent_id>:<rubric_id>`
- `evaluated_component_name`: `<agent_id>`
- `eval_data.parent_run_id`: parent performance row `run_id`
- deterministic `run_id`: `judge:<parent_run_id>:<rubric_id>:<judge_model_id>`

## Threat Model

| Threat | Mitigation |
|---|---|
| Dashboard keeps showing unjudged rows as `Untitled Evaluation` | Use `EvalType.PERFORMANCE`, `name`, and `evaluated_component_name`; verify through API/dashboard when live services are available. |
| Live judging adds user-facing latency | Worker is async/offline and live judging is disabled by default. |
| Sensitive content is sent to judge models | Mandatory privacy gate and `EVAL_LIVE_ALLOW_CONTENT_READ=false` by default. |
| Suite deterministic ids hide genuine reruns | Git identity includes dirty hash; judge identity is included for LLM-judged rows. |
| Direct parent update SQL becomes brittle | Confine it to one helper with tests and a replacement note for future Agno update APIs. |
| Historical rows are accidentally changed | Explicit non-goal; implementation only creates new rows and updates current parent status metadata. |

## Rollback

No DB migration is required.

Rollback:

- revert live recorder row semantics in `agentos/eval_recorder.py`
- remove `agentos/eval_rubrics.py`, `agentos/eval_live_policy.py`, and `agentos/live_judge.py`
- remove the `live-judge` subcommand from `agentos/__main__.py`
- revert deterministic suite identity changes in `evals/conftest.py`
- leave historical rows untouched
