# Plan 15-01 — Fix daily-brief date-mismatch bug (MON-01)

## Objective

Fix the date-mismatch bug where `daily_brief` misses monitor-filed items when the monitor ran the previous evening. Add regression tests that reproduce the original bug and confirm the fix.

**Root cause:** `_read_inbox_items` globs `{day.isoformat()}-*.md` using `date.today()` at brief-run time. If monitor filed items on a different calendar day, they are silently dropped.

</read_first>

Edit `ultra_brain/brief.py:_read_inbox_items`. Add `lookback_days: int = 2` parameter. Replace the single glob with a loop over `range(lookback_days)`:

```python
from datetime import date, timedelta

def _read_inbox_items(vault_root: Path, *, day: date, lookback_days: int = 2) -> list[dict]:
    inbox_dir = vault_root / "Inbox"
    if not inbox_dir.exists():
        return []
    items = []
    seen_paths: set[Path] = set()
    for offset in range(lookback_days):
        check_day = day - timedelta(days=offset)
        for path in sorted(inbox_dir.glob(f"{check_day.isoformat()}-*.md")):
            if path in seen_paths:
                continue
            seen_paths.add(path)
            text = path.read_text(encoding="utf-8")
            lines = text.splitlines()
            title = lines[0].lstrip("# ").strip() if lines else path.stem
            url = published = ""
            for line in lines[1:]:
                if line.startswith("source::"):
                    url = line.split("::", 1)[1].strip()
                elif line.startswith("published::"):
                    published = line.split("::", 1)[1].strip()
            items.append({"title": title, "url": url, "published": published})
    return items
```

The `daily_brief` call to `_read_inbox_items` passes no `lookback_days` argument — the default of 2 applies automatically.
</action>
