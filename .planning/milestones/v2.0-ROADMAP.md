## Milestone Summary

**Key Decisions:**
- `db_id="ultra-brain-main"` shared by all 5 agents (Phase 10 DIAG-02) — simpler than per-agent isolation for a single-owner system
- Post-run wrapper pattern for eval recording (no Agno subclass) — avoids internal coupling
- `performance` vs `accuracy` eval row semantics (Phase 12-04 corrective) — live telemetry = performance parents; suite = accuracy rows
- Stacked `@approval + @tool(requires_confirmation=True)` was the root cause of missing approval rows (Phase 14)
- `psycopg3` (not `psycopg2`) for `check_surfaces.py` — project uses psycopg[binary]>=3.2

**Issues Resolved:**
- AgentOS Memory tab showing 0 entries despite agent runs
- AgentOS Evals tab empty; live runs not recording eval rows
- AgentOS Knowledge tab 404 (no pgvector tables populated)
- AgentOS Approvals tab empty; @approval decorator stacking bug
- Daily-brief date-mismatch (monitor-filed items missed within same day)
- Vault sync --delete wiping VPS-generated inbox items
- check_surfaces.py: psycopg2/psycopg3 mismatch + SQLAlchemy DSN + missing `ai.` schema prefix

**Tech Debt Forward (10 items):**
- Phase 12: `model_id`/`model_provider` null on live eval rows; only test_curator.py wired to suite
- Phase 13: vault reindex is manual CLI, not auto-deploy-triggered; no CI integration test
- Phase 14: `tool_args` logged as type+length summary (privacy guard), not raw values
- Phase 15: `check_surfaces.py` local-only (VPS requires SSH); approvals may be 0 on quiet VPS
